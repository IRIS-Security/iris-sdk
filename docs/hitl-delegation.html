HITL & Delegation

Human-in-the-loop review for high-risk actions, plus user delegation so agents act on behalf of authenticated users — both enforced by Cedar at runtime.

iris hitl — human-in-the-loop review

When a policy or condition rule requires human approval, IRIS queues the request until a reviewer acts. Approved calls proceed; rejected calls raise IrisViolationError. Every decision is logged to the Evidence Vault.

Setup and configuration

iris hitl setup --agent loan-processor
iris hitl config --agent loan-processor
iris hitl rules --agent loan-processor
iris hitl test --agent loan-processor

setup is an interactive wizard: enable HITL, set timeout policy, and declare condition rules (e.g. loan_amount > 50000). HITL fires only when a declared rule matches.

Risk-tiered HITL — gate the critical, not everything

Every policy decision resolves to one of three risk tiers. Approval fatigue — gating every action — trains reviewers to rubber-stamp, which defeats the point of human oversight. A well-configured agent should trigger HITL for a small minority of calls.

TierMeaningHuman involved?
AUTO_ALLOWReversible, low-impact, within declared intent and policy. The default — this should be the large majority of calls.No
STEP_UPIrreversible or high-impact per criteria you declare below. Permitted, pending approval.Yes
AUTO_DENYA clear policy violation (e.g. an absolute compliance block). Blocked without bothering a human.No

Nothing here is hard-coded by IRIS — you declare what's critical for your agent in hitl: under passport.yaml. Every knob defaults to empty/off, so declaring HITL config never changes behavior until you say what should step up:

spec:
  hitl:
    enabled: true
    # Violation severities that step up instead of a silent log or auto-deny
    required_for_risk_levels: [CRITICAL]
    # Action verbs that step up in staging/production (never in dev/test)
    step_up_actions: [write, delete]
    # Data classifications that always step up, regardless of action
    sensitive_data_classifications: [phi, pii]
    # Step up when an action drifts outside the agent's declared intent
    step_up_on_intent_drift: true
    # Arbitrary conditions (existing mechanism, still supported)
    condition_rules:
      - condition: "loan_amount > 50000"
        reason: "High-value loan"

Run iris hitl rules --agent <name> to see exactly which of these are configured and why a specific call would (or wouldn't) trigger review — every triggered review's reason is also recorded on the review itself and visible via iris hitl show <review-id>.

Review queue

iris hitl list
iris hitl list --agent loan-processor
iris hitl list --status all
iris hitl show <review-id>

Approve, reject, escalate

iris hitl approve <review-id> --note "Verified by compliance"
iris hitl reject <review-id> --reason "Insufficient documentation"
iris hitl escalate <review-id> --to senior@company.com

Set IRIS_USER_EMAIL to identify the reviewer. In staging and production, frontier models and high-risk actions may require HITL per rule IRIS-MODEL-005.

SDK integration

After iris hitl approve, pass the approval to drop-in clients:

from iris_anthropic import IrisAnthropic

client = IrisAnthropic(
  passport=passport,
  hitl_approved=True,
)

iris delegation — user delegation

Agents act on behalf of users. Cedar policies evaluate acting_for_user, consent, and scope at runtime.

iris delegation status --agent loan-processor
iris delegation test --agent loan-processor --user alice@company.com
iris delegation log --agent loan-processor

iris users — reviewer management

iris users add --email alice@company.com --role reviewer
iris users list
iris users remove --email alice@company.com

iris dlp — content inspection

Scan files and test prompts against data-loss-prevention rules before they reach an LLM.

iris dlp scan --file customer-data.txt
iris dlp test --agent loan-processor --prompt "..."
